01
Instalação e Configuração Base do Samba
⏱️ 20 min · Dificuldade: ⭐⭐
+55 XP
▼
🎯 Instalar Samba, fazer backup do smb.conf e configurar seção [global]
📦 INSTALAÇÃO
$ sudo apt update && sudo apt install -y samba samba-common-bin
$ sudo systemctl enable --now smbd nmbd
# smbd = serviço de arquivos e impressão  nmbd = resolução NetBIOS

$ sudo systemctl status smbd
● smbd.service - Samba SMB Daemon
   Active: active (running)
💾 BACKUP E CONFIGURAÇÃO GLOBAL
# Sempre faça backup antes de editar!
$ sudo cp /etc/samba/smb.conf /etc/samba/smb.conf.bkp

# Editar smb.conf
$ sudo nano /etc/samba/smb.conf

# Seção [global] — configurações gerais
[global]
   workgroup = WORKGROUP
   server string = Servidor Samba Ubuntu %v
   netbios name = FILESERVER
   security = user
   map to guest = bad user
   dns proxy = no
   log file = /var/log/samba/log.%m
   max log size = 1000
   logging = file

# Verificar sintaxe
$ testparm
Loaded services file OK.
Server role: ROLE_STANDALONE
🔥 FIREWALL
$ sudo ufw allow samba
# Libera portas 137,138 UDP e 139,445 TCP automaticamente
$ sudo ufw status | grep Samba
02
Criando Compartilhamento Público
⏱️ 25 min · Dificuldade: ⭐⭐
+60 XP
▼
🎯 Criar um share público acessível por qualquer usuário sem senha
📂 CRIANDO DIRETÓRIO E PERMISSÕES
$ sudo mkdir -p /srv/samba/publico
$ sudo chmod 0777 /srv/samba/publico
# 0777 = leitura/escrita/execução para todos
$ sudo chown nobody:nogroup /srv/samba/publico
# nobody = usuário sem privilégios para acesso anônimo
📝 ADICIONANDO SHARE AO smb.conf
$ sudo nano /etc/samba/smb.conf
# Adicionar ao final do arquivo:

[Publico]
   comment = Compartilhamento Publico
   path = /srv/samba/publico
   browseable = yes
   read only = no
   guest ok = yes
   create mask = 0666
   directory mask = 0777
   force user = nobody
✅ TESTE DE ACESSO
$ testparm && sudo systemctl restart smbd

# Testar com smbclient (como guest)
$ smbclient //localhost/Publico -N
# -N = sem senha (anonymous)
smb: \> ls     <-- listar arquivos
smb: \> put /etc/hostname teste.txt   <-- enviar arquivo
smb: \> exit

# Ver todos os compartilhamentos disponíveis
$ smbclient -L localhost -N
⚠️ Segurança: Shares públicos com escrita são convenientes mas arriscados. Em produção, use apenas para compartilhamentos temporários e monitore o acesso com smbstatus.
03
Compartilhamento Privado com Autenticação
⏱️ 30 min · Dificuldade: ⭐⭐⭐
+70 XP
▼
🎯 Criar share privado com usuários Samba e controle de acesso por grupo
👤 CRIANDO USUÁRIOS SAMBA
# O usuário deve existir no sistema antes de ser criado no Samba
$ sudo useradd -M -s /usr/sbin/nologin sambauser1
# -M = sem home  -s nologin = não pode fazer login shell

$ sudo smbpasswd -a sambauser1
New SMB password: *****
Retype new SMB password: *****
Added user sambauser1.

# Habilitar o usuário (caso esteja desabilitado)
$ sudo smbpasswd -e sambauser1

# Listar usuários Samba
$ sudo pdbedit -L
sambauser1:1001:Samba User 1

# Criar grupo para o compartilhamento
$ sudo groupadd sambashare
$ sudo usermod -aG sambashare sambauser1
📂 DIRETÓRIO E SHARE PRIVADO
$ sudo mkdir -p /srv/samba/privado
$ sudo chown root:sambashare /srv/samba/privado
$ sudo chmod 2770 /srv/samba/privado
# 2770 = setGID + rwx para owner e grupo, nada para outros

# Adicionar ao smb.conf:
[Privado]
   comment = Compartilhamento Privado
   path = /srv/samba/privado
   browseable = no
   read only = no
   guest ok = no
   valid users = @sambashare
   create mask = 0660
   directory mask = 2770
   force group = sambashare
🧪 TESTANDO AUTENTICAÇÃO
$ testparm && sudo systemctl restart smbd

# Login com usuário Samba
$ smbclient //localhost/Privado -U sambauser1
Password for [WORKGROUP\sambauser1]:
smb: \>

# Ver conexões ativas
$ sudo smbstatus
Samba version 4.x.x
PID  Username  Group      Machine
---  ---------  ---------  -------
1234 sambauser1 sambashare 192.168.1.50
04
Acesso Windows e Montar no Linux
⏱️ 20 min · Dificuldade: ⭐⭐
+55 XP
▼
🎯 Acessar o Samba pelo Windows Explorer e montar compartilhamentos no Linux
🪟 ACESSO VIA WINDOWS EXPLORER
# No Windows, abrir o Explorer e digitar na barra de endereços:
\\192.168.1.10\Publico     # share público
\\192.168.1.10\Privado     # share privado (pedirá login)
\\FILESERVER               # pelo netbios name

# Ou via CMD do Windows:
net use Z: \\192.168.1.10\Publico
net use Y: \\192.168.1.10\Privado /user:sambauser1
🐧 MONTAR SAMBA NO LINUX (cifs-utils)
$ sudo apt install -y cifs-utils

# Montagem manual temporária
$ sudo mkdir /mnt/samba_pub
$ sudo mount -t cifs //192.168.1.10/Publico /mnt/samba_pub -o guest

# Montagem autenticada
$ sudo mount -t cifs //192.168.1.10/Privado /mnt/samba_priv   -o username=sambauser1,password=senha123,uid=1000,gid=1000

# Montar automaticamente no boot via /etc/fstab
//192.168.1.10/Publico  /mnt/samba_pub  cifs  guest,uid=1000  0  0

# Verificar montagens
$ df -h | grep cifs
//192.168.1.10/Publico   20G  5G  15G  25%  /mnt/samba_pub
✅ Checklist Samba: smbd e nmbd ativos · testparm sem erros · smbclient conecta · share público acessível como guest · share privado exige autenticação · Windows consegue acessar o servidor
