🌐 Configurando a Rede com Netplan
💡 O que é o Netplan?
O Netplan é a ferramenta do Ubuntu para configurar interfaces de rede. O arquivo de configuração fica em /etc/netplan/ e usa o formato YAML — que exige recuo (indentação) com espaços.

Figura: Placa NAT usa DHCP · Placa Interna usa IP fixo classe C

Esta configuração é usada quando a placa está conectada em NAT — o roteador/VirtualBox atribui o IP automaticamente.
1
Abra o arquivo netplan para editar:
sudo nano /etc/netplan/00-installer-config.yaml
2
Cole o conteúdo abaixo — respeite exatamente os espaços!
# /etc/netplan/00-installer-config.yaml network: version: 2 ethernets: enp0s3: dhcp4: true
3
Salve: Ctrl+X → Y → Enter
4
Aplique: sudo netplan apply
5
Verifique o IP recebido: ip addr show enp0s3
✅ Resultado esperado: você verá uma linha inet 10.0.2.15/24 (ou similar) — isso significa que o DHCP funcionou!
Esta configuração é para a placa conectada em Rede Interna. O IP fica fixo e as outras VMs da rede interna apontarão para este endereço como gateway.
1
Abra o arquivo netplan:
sudo nano /etc/netplan/00-installer-config.yaml
2
Configure o IP fixo 192.168.10.1 com máscara /24 (classe C):
# /etc/netplan/00-installer-config.yaml network: version: 2 ethernets: enp0s8: dhcp4: false addresses: - 192.168.10.1/24
⚠️ Atenção à indentação! Cada nível usa 2 espaços — nunca Tab. O YAML é sensível a isso. Se errar, o netplan apply vai mostrar um erro.
3
Aplique: sudo netplan apply
4
Verifique: ip addr show enp0s8 — deve aparecer inet 192.168.10.1/24
✅ As outras VMs da Rede Interna devem usar 192.168.10.1 como gateway e DNS.
Esta é a configuração completa: enp0s3 via DHCP (saída para internet via NAT) e enp0s8 com IP fixo (rede interna das VMs).

Figura: Firewall com duas placas — NAT (internet) e Rede Interna (VMs)

1
Abra o arquivo netplan:
sudo nano /etc/netplan/00-installer-config.yaml
2
Veja os nomes das suas interfaces primeiro: ip link show — procure por enp0s3 e enp0s8 (ou similares)
3
Configuração completa com as duas placas:
# /etc/netplan/00-installer-config.yaml # ATENÇÃO: use ESPAÇOS, nunca Tab! network: version: 2 ethernets: enp0s3: # Placa 1 — NAT (internet) dhcp4: true # IP automático enp0s8: # Placa 2 — Rede Interna dhcp4: false # IP fixo addresses: - 192.168.10.1/24 # Classe C — gateway
📐 Como ler a indentação:

network:        ← nível 0 (sem espaço)
··version: 2   ← nível 1 (2 espaços)
··ethernets:   ← nível 1 (2 espaços)
····enp0s3:    ← nível 2 (4 espaços)
······dhcp4:   ← nível 3 (6 espaços)
4
Salve: Ctrl+X → Y → Enter
5
Aplique a configuração: sudo netplan apply
6
Verifique as duas interfaces:
ip addr show enp0s3 → deve ter IP tipo 10.0.2.x
ip addr show enp0s8 → deve ter 192.168.10.1/24
✅ Se ambas apareceram com IP, a rede está configurada! Agora pode ir para a próxima etapa.

⚡ Habilitando o IP Forwarding
💡 O que é IP Forwarding?
Permite que o Ubuntu "repasse" pacotes entre as interfaces. Sem isso, a VM não funciona como roteador/gateway — os pacotes chegam pela Rede Interna mas não saem pela NAT (e vice-versa).
1
Edite o arquivo de configuração do kernel:
sudo nano /etc/sysctl.conf
2
Procure a linha abaixo e remova o # do início (ou adicione se não existir):
# Habilitar encaminhamento de pacotes IPv4 net.ipv4.ip_forward=1
3
Aplique sem reiniciar: sudo sysctl -p
4
Verifique se está ativo (deve mostrar 1):
cat /proc/sys/net/ipv4/ip_forward
✅ Saída esperada: 1 — isso confirma que o forwarding está ativo.

🔥 Configurando o Firewall UFW
💡 O que é o UFW?
O UFW (Uncomplicated Firewall) é a ferramenta do Ubuntu para controlar o firewall de forma simples. Ele decide quais conexões entram e saem da máquina com comandos fáceis.

Figura: O UFW filtra o tráfego antes de chegar ao servidor

🔐 Passo a passo — Configurar o UFW do zero
🚨 CUIDADO ANTES DE TUDO: Se você vai acessar via SSH, libere a porta 22 ANTES de ativar o UFW. Senão você perde o acesso à máquina!
1
Libere o SSH primeiro (porta 22) — isso é obrigatório!
sudo ufw allow 22/tcp
# PASSO 1 — Libere SSH ANTES de qualquer coisa $ sudo ufw allow 22/tcp Rules updated Rules updated (v6) # PASSO 2 — Defina política padrão: bloquear tudo que entra $ sudo ufw default deny incoming $ sudo ufw default allow outgoing # PASSO 3 — Ative o UFW $ sudo ufw enable Command may disrupt existing ssh connections. Proceed with operation (y|n)? y Firewall is active and enabled on system startup
2
Libere outras portas conforme necessário:
# Liberar HTTP e HTTPS (servidores web) $ sudo ufw allow 80/tcp $ sudo ufw allow 443/tcp # Bloquear Telnet (inseguro — sem criptografia) $ sudo ufw deny 23/tcp # Bloquear FTP (inseguro) $ sudo ufw deny 21/tcp # Bloquear RDP — acesso remoto Windows $ sudo ufw deny 3389/tcp
3
Ver o status — verifique todas as regras:
# Ver status básico $ sudo ufw status # Ver com mais detalhes (recomendado) $ sudo ufw status verbose # Ver regras numeradas (para remover depois) $ sudo ufw status numbered
✅ O que você verá:

Status: active
To             Action     From
22/tcp        ALLOW IN   Anywhere
80/tcp        ALLOW IN   Anywhere
23/tcp        DENY IN    Anywhere
4
Remover uma regra se errar:
# Ver regras com número $ sudo ufw status numbered # [ 1] 22/tcp ALLOW IN Anywhere # [ 2] 80/tcp ALLOW IN Anywhere # Deletar a regra número 2 $ sudo ufw delete 2 # Recarregar se necessário $ sudo ufw reload
🔌 Portas que você precisa conhecer
Porta Serviço UFW Por quê?
22/tcp SSH ✅ ALLOW Acesso remoto seguro
80/tcp HTTP ✅ ALLOW Web sem criptografia
443/tcp HTTPS ✅ ALLOW Web com criptografia (seguro)
23/tcp Telnet 🚫 DENY Sem criptografia — tudo em texto claro
21/tcp FTP 🚫 DENY Transferência de arquivo insegura
3389/tcp RDP 🚫 DENY Área de trabalho remota Windows
3306/tcp MySQL ⚠️ CUIDADO Só abrir se necessário e com restrição
