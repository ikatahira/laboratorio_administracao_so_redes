CONCEITOS — O QUE É O UFW?
Base teórica
💡 UFW (Uncomplicated Firewall) é a interface simplificada para o iptables no Ubuntu. Ele gerencia regras de entrada, saída e encaminhamento de pacotes de rede.
🔥 UFW

Uncomplicated Firewall — interface amigável para o iptables. Controla quais conexões são permitidas ou bloqueadas na VM.

🚫 Deny / Allow

As duas ações básicas do UFW: allow libera o tráfego em uma porta; deny bloqueia silenciosamente.

⚡ IP Forwarding

Habilita o kernel a encaminhar pacotes entre interfaces de rede — essencial para a VM funcionar como gateway/roteador.

🌐 Masquerading

Tipo de NAT: traduz o IP interno das VMs pelo IP público do firewall, permitindo que elas acessem a internet.

📋 COMANDOS UFW ESSENCIAIS
# Ver status do firewall $ sudo ufw status verbose # Ativar / desativar UFW $ sudo ufw enable $ sudo ufw disable # Liberar e bloquear portas $ sudo ufw allow 22/tcp # ALLOW — libera porta 22 TCP $ sudo ufw deny 23/tcp # DENY — bloqueia Telnet # Definir políticas padrão $ sudo ufw default deny incoming # Bloquear tudo que entra $ sudo ufw default allow outgoing # Permitir tudo que sai # Recarregar e ver regras numeradas $ sudo ufw reload $ sudo ufw status numbered # Remover uma regra específica pelo número $ sudo ufw delete 3

🎯 Objetivo: Instalar o Ubuntu Server em uma VM dedicada com duas NICs, habilitar IP Forwarding e masquerading, e aplicar regras UFW para que ela atue como firewall/gateway da rede interna.
🔥 Por que um Firewall como VM separada?
Em arquiteturas reais, o firewall é um dispositivo dedicado posicionado entre a rede interna e a internet. Neste laboratório, simulamos isso com uma VM que tem duas placas de rede: uma para a internet (NAT) e outra para a rede interna das VMs. Todo o tráfego passa por ela — e ela decide o que deixa passar.

🔧 Placa de rede não funcionou após instalar? Siga estes passos:

✅ 1. Ativar a interface manualmente

sudo ip link set enp0s3 up

✅ 2. Solicitar IP via DHCP

🧪 3. Testar conectividade

⚠️ Atenção: Substitua enp0s3 pelo nome real da sua interface. Use ip link show para verificar.
⚠️ O firewall precisa de IP estático na interface interna para ser o gateway fixo das outras VMs. Use o netplan para configurar.
1

Identifique os nomes das interfaces de rede:

# Ver todas as interfaces e seus nomes seunomefirewall@seunomefirewall:~$ ip link show 1: lo: <LOOPBACK> ... 2: enp0s3: <BROADCAST> ... ← Adaptador 1 (NAT / internet) 3: enp0s8: <BROADCAST> ... ← Adaptador 2 (Rede Interna / LAN)
2

Edite o arquivo de configuração do netplan:

seunomefirewall@seunomefirewall:~$ sudo nano /etc/netplan/00-installer-config.yaml
3

Substitua o conteúdo pelo seguinte (use espaços, nunca tabs — YAML é sensível à indentação):

# /etc/netplan/00-installer-config.yaml network: version: 2 ethernets: enp0s3: dhcp4: true # NAT — recebe IP do VirtualBox automaticamente enp0s8: dhcp4: false # LAN interna — IP fixo addresses: - 192.168.56.1/24 # IP do firewall na rede interna (gateway das VMs) nameservers: addresses: [8.8.8.8, 1.1.1.1]
4

Aplique as configurações e verifique:

seunomefirewall@seunomefirewall:~$ sudo netplan apply seunomefirewall@seunomefirewall:~$ ip addr show enp0s8 3: enp0s8: <BROADCAST,MULTICAST,UP> inet 192.168.56.1/24 brd 192.168.56.255 scope global enp0s8
5

Nas VMs Desktop e Server, configure o gateway como 192.168.56.1 no netplan delas.

✅ Se o IP 192.168.56.1 apareceu na interface enp0s8, a rede interna está configurada corretamente.
💡 IP Forwarding permite que o kernel Linux encaminhe pacotes entre interfaces. Sem ele, o tráfego da LAN não chega à internet — o firewall simplesmente descartaria os pacotes.
1

Habilitar IP Forwarding permanentemente no kernel:

# Editar o arquivo de configuração do kernel seunomefirewall@seunomefirewall:~$ sudo nano /etc/sysctl.conf # Encontrar e descomentar (remover o #) a linha: net.ipv4.ip_forward=1 # Aplicar sem reiniciar seunomefirewall@seunomefirewall:~$ sudo sysctl -p net.ipv4.ip_forward = 1 # Confirmar que está ativo (deve retornar 1) seunomefirewall@seunomefirewall:~$ cat /proc/sys/net/ipv4/ip_forward 1
2

Configurar o masquerading (NAT) no UFW para que as VMs da LAN acessem a internet usando o IP do firewall:

# Editar as regras brutas do UFW (antes do *filter) seunomefirewall@seunomefirewall:~$ sudo nano /etc/ufw/before.rules # Adicionar no INÍCIO do arquivo, antes de *filter: # NAT — masquerading para saída pela interface enp0s3 (internet) *nat :POSTROUTING ACCEPT [0:0] -A POSTROUTING -s 192.168.56.0/24 -o enp0s3 -j MASQUERADE COMMIT
3

Permitir o encaminhamento (forwarding) no arquivo de configuração do UFW:

seunomefirewall@seunomefirewall:~$ sudo nano /etc/default/ufw # Alterar a linha DEFAULT_FORWARD_POLICY: DEFAULT_FORWARD_POLICY="ACCEPT"
✅ Com o IP Forwarding ativo e o masquerading configurado, as VMs da LAN já conseguem acessar a internet pelo firewall — desde que as regras UFW permitam.
⚠️ Regra de ouro: sempre libere o SSH antes de ativar o UFW. Se esquecer, perde o acesso remoto à VM sem possibilidade de recuperação pelo terminal.
🔐 PASSO 1 — Liberar SSH e ativar UFW
seunomefirewall@seunomefirewall:~$ sudo ufw allow 22/tcp Rules updated seunomefirewall@seunomefirewall:~$ sudo ufw enable Command may disrupt existing ssh connections. Proceed with operation (y|n)? y Firewall is active and enabled on system startup
🛡️ PASSO 2 — Políticas padrão
seunomefirewall@seunomefirewall:~$ sudo ufw default deny incoming Default incoming policy changed to 'deny' seunomefirewall@seunomefirewall:~$ sudo ufw default allow outgoing Default outgoing policy changed to 'allow' seunomefirewall@seunomefirewall:~$ sudo ufw default allow routed Default routed policy changed to 'allow' # ↑ essencial para o masquerading funcionar
✅ PASSO 3 — Permitir serviços essenciais
seunomefirewall@seunomefirewall:~$ sudo ufw allow 22/tcp # SSH (já liberado, reforçando) seunomefirewall@seunomefirewall:~$ sudo ufw allow 80/tcp # HTTP seunomefirewall@seunomefirewall:~$ sudo ufw allow 443/tcp # HTTPS
🚫 PASSO 4 — Bloquear serviços inseguros
seunomefirewall@seunomefirewall:~$ sudo ufw deny 23/tcp # Telnet — sem criptografia seunomefirewall@seunomefirewall:~$ sudo ufw deny 21/tcp # FTP — senhas em texto claro seunomefirewall@seunomefirewall:~$ sudo ufw deny 3389/tcp # RDP — alvo frequente de ataques
🔍 PASSO 5 — Recarregar e verificar status
seunomefirewall@seunomefirewall:~$ sudo ufw reload Firewall reloaded seunomefirewall@seunomefirewall:~$ sudo ufw status verbose Status: active Logging: on (low) Default: deny (incoming), allow (outgoing), allow (routed) To Action From -- ------ ---- 22/tcp ALLOW IN Anywhere 80/tcp ALLOW IN Anywhere 443/tcp ALLOW IN Anywhere 23/tcp DENY IN Anywhere 21/tcp DENY IN Anywhere 3389/tcp DENY IN Anywhere
🧪 PASSO 6 — Testar conectividade nas VMs clientes
# Nas outras VMs, verificar que o gateway está correto seunomecliente@seunomecliente:~$ ip route show default via 192.168.56.1 dev enp0s8 ← tráfego passa pelo firewall # Testar acesso à internet através do firewall seunomecliente@seunomecliente:~$ ping -c 3 8.8.8.8 3 packets transmitted, 3 received, 0% packet loss # Testar resolução de nomes seunomecliente@seunomecliente:~$ ping -c 3 google.com 3 packets transmitted, 3 received, 0% packet loss
✅ Se o ping funciona nas outras VMs, o masquerading e o IP Forwarding estão funcionando. O tráfego está passando pelo firewall conforme esperado.
