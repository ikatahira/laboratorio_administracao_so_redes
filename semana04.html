📘 Glossário Rápido
VLAN
Virtual LAN — segmenta logicamente a rede sem hardware adicional, isolando tráfego por função/departamento.
Trunk Port
Porta que carrega tráfego de múltiplas VLANs usando tags IEEE 802.1Q. Conecta switches entre si.
NAT / PAT
Traduz IPs privados para público. PAT (Port Address Translation) mapeia múltiplos hosts num único IP externo.
ACL
Access Control List — filtra pacotes por origem/destino/protocolo. Aplica política de segurança no roteador.
STP
Spanning Tree Protocol — evita loops de layer 2 bloqueando portas redundantes automaticamente.
WPA3
Protocolo de segurança Wi-Fi mais recente. SAE substitui PSK, oferecendo resistência a ataques de dicionário.

Missões Práticas

até +240 XP
M1
Configuração de VLANs no Switch
🔀 Switches · ⏱ 30 min · Dificuldade: Médio
+60 XP ▼

Criar 3 VLANs (Administração, Vendas, TI) e configurar portas de acesso e trunk.

1

Acesse o switch via console e entre em modo privilegiado com enable, depois configure terminal.

2

Crie as três VLANs no switch:

Switch(config)# vlan 10
Switch(config-vlan)# name Administracao
Switch(config-vlan)# exit
Switch(config)# vlan 20
Switch(config-vlan)# name Vendas
Switch(config-vlan)# exit
Switch(config)# vlan 30
Switch(config-vlan)# name TI
3

Atribua porta Fa0/1 à VLAN 10 (Administração) como access port:

Switch(config)# interface FastEthernet 0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10
! Repita para Fa0/2 (VLAN 20) e Fa0/3 (VLAN 30)
4

Configure a porta Gi0/1 como trunk para o roteador (router-on-a-stick):

Switch(config)# interface GigabitEthernet 0/1
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan 10,20,30
5

Verifique com show vlan brief e show interfaces trunk.

M2
Setup de Roteador Básico
📡 Roteadores · ⏱ 30 min · Dificuldade: Médio
+60 XP ▼

Configurar IP nas interfaces, roteamento estático entre duas redes e NAT.

1

Acesse o roteador via console. Configure as subinterfaces para inter-VLAN routing (router-on-a-stick):

Router(config)# interface GigabitEthernet 0/0
Router(config-if)# no shutdown

Router(config)# interface GigabitEthernet 0/0.10
Router(config-subif)# encapsulation dot1Q 10
Router(config-subif)# ip address 192.168.10.1 255.255.255.0

Router(config)# interface GigabitEthernet 0/0.20
Router(config-subif)# encapsulation dot1Q 20
Router(config-subif)# ip address 192.168.20.1 255.255.255.0
2

Configure a interface WAN (Gi0/1) com IP público e habilite NAT:

Router(config)# interface GigabitEthernet 0/1
Router(config-if)# ip address 200.10.5.1 255.255.255.0
Router(config-if)# ip nat outside
Router(config-if)# no shutdown

Router(config)# interface GigabitEthernet 0/0.10
Router(config-subif)# ip nat inside

Router(config)# ip nat inside source list 1 interface Gi0/1 overload
Router(config)# access-list 1 permit 192.168.0.0 0.0.255.255
3

Verifique com show ip route, show ip nat translations e faça ping entre VLANs.

M3
Configuração de Wi-Fi Seguro (WPA3)
📶 Access Point · ⏱ 25 min · Dificuldade: Fácil
+50 XP ▼

Configurar SSID, senha forte, canal adequado e segurança WPA3 no access point.

1

Acesse a interface web do AP (tipicamente 192.168.1.1 ou 192.168.0.1) com admin/admin.

2

Configure as redes SSIDs — uma para cada VLAN/departamento:

── SSID Corporativo ───────────────────────────
Nome (SSID) : REDES-CORP-S4
Segurança : WPA3-SAE (ou WPA2/WPA3 Mixed para compatibilidade)
Senha : Ft3ch@2025!Seguro (mín 12 chars, maiúscula, número, símbolo)
Banda : 5GHz (menos interferência)
Canal : Auto / 36 (canais não sobrepostos: 36,40,44,48...)
VLAN tag : 10 (VLAN Administração)

── SSID Visitantes ────────────────────────────
Nome (SSID) : REDES-GUEST-S4
Segurança : WPA2-PSK
Isolamento : Habilitado (impede acesso entre clientes)
VLAN tag : 99 (VLAN isolada)
3

Desative o WPS (vulnerável a ataques de força bruta) e oculte o SSID de visitantes se necessário.

4

Teste a conexão a partir de um dispositivo e confirme que o IP atribuído pertence à sub-rede correta.

M4
Integração Completa — Topologia Funcional
🔗 Lab Final · ⏱ 35 min · Dificuldade: Avançado
+70 XP ▼

Integrar switch com VLANs, roteador e access point numa topologia totalmente funcional.

1

Confirme que o switch tem as VLANs 10, 20 e 30 criadas (Missão 1) e o trunk para o roteador ativo.

2

Confirme que o roteador tem as subinterfaces .10, .20 e .30 com IPs gateway para cada VLAN.

3

Conecte o access point a uma porta do switch na VLAN de gerência. Configure o IP do AP na mesma sub-rede do gateway.

4

Execute o checklist de verificação:

── Checklist de Verificação ───────────────────
[ ] ping 192.168.10.1 → Gateway VLAN Admin responde?
[ ] ping 192.168.20.1 → Gateway VLAN Vendas responde?
[ ] ping 192.168.30.1 → Gateway VLAN TI responde?
[ ] ping 8.8.8.8 → NAT/WAN funciona? (internet)
[ ] Wi-Fi conectado entrega IP da VLAN correta?
[ ] VLANs isoladas entre si? (testar ping cross-VLAN)
5

Documente a topologia (esboço com IPs, VLANs, interfaces) e salve a configuração: copy running-config startup-config.
