O QUE É O GUFW?
Base teórica
💡 GUFW (Graphical Uncomplicated Firewall) é a interface gráfica oficial do UFW para ambientes desktop Linux. Ele permite gerenciar regras de firewall visualmente — ativar/desativar o firewall, criar regras de allow/deny e definir políticas de entrada e saída, tudo com cliques, sem usar o terminal.
🛡️ GUFW

Front-end gráfico do UFW. Usa GTK e roda no GNOME do Ubuntu Desktop. Internamente chama os mesmos comandos ufw do terminal.

🔧 Back-end UFW

Toda regra criada no GUFW é traduzida para comandos UFW. O que você clica na tela equivale exatamente ao que digitaria no terminal.

🖥️ Ubuntu Desktop

O GUFW é indicado para o ambiente Desktop — onde existe interface gráfica. No Server (sem GUI), usa-se o UFW diretamente pelo terminal.

🔒 Profiles

O GUFW permite criar perfis (Home, Office, Public) com políticas diferentes para cada ambiente de rede — troca com um clique.

⚠️ O GUFW exige privilégios de administrador (sudo) para ser executado. Se aberto sem sudo, abrirá somente leitura — não será possível alterar nenhuma regra.
📦
INSTALAR O GUFW NO UBUNTU DESKTOP
+10 XP
💡 O GUFW pode não vir instalado por padrão no Ubuntu Desktop 22.04. A instalação é simples e rápida — apenas um comando no terminal.
📋 PASSO A PASSO — INSTALAÇÃO
1

Abra o terminal (Ctrl + Alt + T) e atualize a lista de repositórios:

usuario@ubuntu:~$ sudo apt update Hit:1 http://br.archive.ubuntu.com/ubuntu jammy InRelease Reading package lists... Done
2

Instale o GUFW:

usuario@ubuntu:~$ sudo apt install gufw -y Reading package lists... Done Building dependency tree... Done The following NEW packages will be installed: gufw 0 upgraded, 1 newly installed, 0 to remove and 0 not upgraded. Do you want to continue? [Y/n] Y Setting up gufw ...
3

Abra o GUFW com privilégios de administrador:

# Opção 1 — pelo terminal (recomendado para garantir permissões) usuario@ubuntu:~$ sudo gufw # Opção 2 — pelo menu do GNOME # Atividades → digitar "Firewall" → clicar em "Configuração do Firewall" # O sistema solicitará a senha de administrador automaticamente
4

Verifique que o UFW está instalado (backend do GUFW):

usuario@ubuntu:~$ sudo ufw version ufw 0.36.1 Copyright 2008-2021 Canonical Ltd.
✅ Com o GUFW instalado e aberto com sudo, a janela principal do firewall aparecerá. O próximo passo é entender a interface.

🔍 Elementos da janela:
  • Toggle Status — liga/desliga o firewall com um clique
  • Perfil — seleciona o conjunto de regras ativo (Home, Office, Public)
  • Política de Entrada/Saída — define o comportamento padrão para tráfego não coberto por regras específicas
  • Lista de Regras — exibe todas as regras ativas; clique em uma para selecioná-la e deletar
  • Botão + (Adicionar) — abre o diálogo para criar uma nova regra
💡 Os Perfis permitem ter conjuntos de regras diferentes para cada ambiente de rede. Ideal para laptops que transitam entre casa, trabalho e redes públicas.
🏠 Home

Rede doméstica confiável. Política mais permissiva — entrada liberada para serviços locais, saída totalmente liberada.

🏢 Office

Rede corporativa. Regras específicas para aplicativos de trabalho. Geralmente bloqueia serviços desnecessários.

☕ Public

Redes públicas (cafés, aeroportos). Política mais restritiva — nega toda entrada, só permite conexões iniciadas pela máquina.

📋 COMO TROCAR DE PERFIL
No GUFW:
1. Abra o GUFW com sudo gufw
2. No menu superior, clique em Perfil
3. Selecione o perfil desejado (Home / Office / Public)
4. As regras e políticas mudarão automaticamente para as daquele perfil

Criar um novo perfil:
Menu Editar → Preferências → Perfis → clique em + e nomeie o novo perfil
⚠️ Cada perfil armazena suas próprias regras independentemente. Trocar de perfil não apaga as regras dos outros — elas ficam salvas para quando você voltar.
💡 As Políticas Padrão definem o que acontece com tráfego que não corresponde a nenhuma regra específica. São o comportamento de fallback do firewall.
Política Entrada (Incoming) Saída (Outgoing) Uso recomendado
🏠 Home Allow Allow Rede doméstica totalmente confiável
🔒 Padrão seguro Deny Allow Recomendado para servidores e uso geral
☕ Public Deny Deny Redes públicas desconhecidas
🧪 Reject Reject Allow Bloqueia e informa o remetente (diferente do Deny silencioso)
🖱️ COMO ALTERAR NO GUFW
Na tela principal do GUFW, os menus dropdown Entrada e Saída controlam as políticas padrão.
Opções disponíveis: Allow · Deny · Reject

A diferença entre Deny e Reject:
• Deny — descarta o pacote silenciosamente (o remetente não recebe resposta)
• Reject — descarta e envia uma mensagem de erro ao remetente (ICMP unreachable)
💡 Para adicionar uma regra no GUFW, clique no botão + na barra inferior da janela. Um diálogo com três abas aparecerá: Simples, Avançado e App.
🖱️ ABA "SIMPLES" — O MAIS USADO NO DIA A DIA
Campos disponíveis:
• Política: Allow / Deny / Reject / Limit
• Direção: In / Out / Both
• Protocolo: TCP / UDP / Both
• Porta: número da porta (ex: 22, 80, 443)

Exemplos de regras simples:
Allow · In · TCP · 22 → libera SSH
Deny · In · TCP · 23 → bloqueia Telnet
Allow · In · Both · 80 → libera HTTP (TCP e UDP)
🖱️ ABA "AVANÇADO" — CONTROLE POR IP
Permite filtrar por IP de origem e destino:
• From (origem): IP específico ou faixa CIDR (ex: 192.168.1.0/24)
• To (destino): IP de destino da regra
• Port (origem/destino): porta da origem e da regra

Exemplos avançados:
Allow · In · TCP · From: 192.168.1.100 · Port: 22 → só esse IP acessa SSH
Deny · In · TCP · From: 10.0.0.0/8 · Port: 3389 → bloqueia RDP de toda a faixa 10.x
🖱️ ABA "APP" — REGRAS PRÉ-DEFINIDAS
O UFW possui perfis de aplicativos pré-configurados:
• OpenSSH → abre porta 22/tcp
• Apache → abre portas 80 e 443
• Nginx → abre portas 80 e 443
• Samba → abre as portas do compartilhamento de arquivos

Selecione o aplicativo, defina Allow/Deny e clique em Adicionar — mais simples do que lembrar cada porta.
✅ Toda regra criada no GUFW é persistente — sobrevive ao reboot automaticamente, pois o UFW salva em /etc/ufw/user.rules.

GUFW vs UFW — QUANDO USAR CADA UM?
Comparativo
Critério 🛡️ GUFW 🔥 UFW (terminal)
Interface Gráfica (cliques) Linha de comando
Ambiente Desktop (GNOME) Server e Desktop
Curva de aprendizado Baixa — intuitivo Média — requer saber os comandos
Automação / Scripts Não suporta Total — ideal para scripts
Conexão remota (SSH) Não funciona sem GUI Funciona em qualquer cenário
Regras por IP Aba Avançado Diretamente no comando
Perfis de rede Home / Office / Public Não tem nativamente
Back-end Ambos usam o mesmo — UFW → iptables/nftables
🎯 Resumo prático:
Use o GUFW no seu desktop pessoal ou nos computadores de usuários comuns — é mais acessível e visual.
Use o UFW pelo terminal em servidores, via SSH, em scripts de automação ou quando precisar de controle preciso e repetível.
