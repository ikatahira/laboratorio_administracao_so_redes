📖 GLOSSÁRIO TÉCNICO
cron
Daemon que executa tarefas agendadas baseadas em expressões de tempo (minuto hora dia mês dia_semana)
anacron
Complemento ao cron para sistemas que ficam desligados — executa tarefas perdidas na próxima inicialização
crontab
Arquivo de tabela de tarefas cron — um por usuário em /var/spool/cron/crontabs/
trap
Comando Bash que captura sinais e eventos (ERR, EXIT, INT) para tratamento de erros
awk
Linguagem de processamento de texto por campos — poderosa para extrair e transformar dados
heredoc
Sintaxe << EOF ... EOF para passar blocos de texto multilinha como entrada

🎯 Usar arrays e ferramentas awk/sed/grep para processar dados do sistema
📊 ARRAYS EM BASH
#!/bin/bash
# Arrays indexados
SERVICOS=("ssh" "apache2" "mysql" "nginx" "cron")

echo "Total de serviços: ${#SERVICOS[@]}"
echo "Primeiro: ${SERVICOS[0]}"
echo "Último  : ${SERVICOS[-1]}"

# Iterar array
echo "=== Status dos Serviços ==="
for SVC in "${SERVICOS[@]}"; do
    STATUS=$(systemctl is-active "$SVC" 2>/dev/null || echo "não instalado")
    printf "  %-12s → %s
" "$SVC" "$STATUS"
done

# Arrays associativos (dicionários)
declare -A PORTAS
PORTAS["ssh"]=22
PORTAS["http"]=80
PORTAS["https"]=443
PORTAS["mysql"]=3306

echo ""
echo "=== Portas dos Serviços ==="
for SVC in "${!PORTAS[@]}"; do    # ${!arr} = chaves
    printf "  %-8s → porta %d
" "$SVC" "${PORTAS[$SVC]}"
done
🔤 AWK, SED E GREP NA PRÁTICA
# AWK — extrair campos específicos
$ df -h | awk 'NR>1 {print $1, $5}'
# NR>1 pula o cabeçalho  $1=filesystem $5=uso%

$ ps aux | awk '$3 > 50 {print "CPU alta:", $1, $3"%", $11}'
# Processos com CPU > 50%

# SED — substituição e filtragem
$ sed 's/root/ADMIN/g' /etc/passwd | head -3
$ sed -n '/^www/p' /etc/passwd
# -n suprime output  /padrão/p imprime só o que bate

# GREP com regex extendida
$ grep -E "^[a-z]+:[x]:[0-9]{1,4}:" /etc/passwd
# Usuários com UID até 4 dígitos

$ grep -c "Failed password" /var/log/auth.log
# -c conta ocorrências de tentativas SSH falhas

+70 XP

🎯 Criar scripts robustos com tratamento de erros, trap e sistema de log
🚨 SCRIPT ROBUSTO COM TRATAMENTO DE ERROS
#!/bin/bash
# script_robusto.sh
set -euo pipefail
# set -e  → sai ao primeiro erro
# set -u  → sai se variável indefinida usada
# set -o pipefail → sai se qualquer pipe falhar

# ── Sistema de Log ─────────────────────────
readonly LOG_FILE="/var/log/meu_script.log"
readonly SCRIPT_NAME=$(basename "$0")

log()  { echo "[$(date '+%Y-%m-%d %H:%M:%S')] [INFO]  $*" | tee -a "$LOG_FILE"; }
warn() { echo "[$(date '+%Y-%m-%d %H:%M:%S')] [WARN]  $*" | tee -a "$LOG_FILE" >&2; }
erro() { echo "[$(date '+%Y-%m-%d %H:%M:%S')] [ERRO]  $*" | tee -a "$LOG_FILE" >&2; }

# ── Trap: executar na saída/erro ─────────────
cleanup() {
    local EXIT_CODE=$?
    if [ $EXIT_CODE -ne 0 ]; then
        erro "Script falhou com código $EXIT_CODE na linha $BASH_LINENO"
    else
        log "Script concluído com sucesso"
    fi
}
trap cleanup EXIT    # Sempre executa ao sair

trap_erro() {
    erro "Erro na linha $1 do script"
    exit 1
}
trap 'trap_erro $LINENO' ERR    # Executa em qualquer erro

# ── Validação de entrada ──────────────────
validar_dir() {
    local DIR="$1"
    if [ ! -d "$DIR" ]; then
        erro "Diretório não existe: $DIR"
        return 1
    fi
    if [ ! -w "$DIR" ]; then
        erro "Sem permissão de escrita: $DIR"
        return 1
    fi
    return 0
}

# ── Execução ──────────────────────────────
log "=== Iniciando $SCRIPT_NAME ==="

ALVO="${1:?ERRO: Forneça o diretório alvo como argumento}"
validar_dir "$ALVO"

log "Processando: $ALVO"
# ... resto do script ...
⚠️ BASH_LINENO: Variável especial que contém o número da linha atual. Junto com trap ... ERR, permite identificar exatamente onde o script falhou — essencial em scripts de produção.

Crontab — Agendamento de Tarefas
⏱️ 30 min · Dificuldade: ⭐⭐⭐
+70 XP
▼
🎯 Configurar crontab do usuário e do sistema para executar tarefas automaticamente
📅 SINTAXE DO CRONTAB
# ┌─── Minuto (0-59)
# │ ┌─── Hora (0-23)
# │ │ ┌─── Dia do mês (1-31)
# │ │ │ ┌─── Mês (1-12)
# │ │ │ │ ┌─── Dia da semana (0=Dom, 1=Seg ... 6=Sáb)
# │ │ │ │ │
# * * * * *   comando

# Exemplos:
0 * * * *       /script.sh   # Todo início de hora
0 6 * * *       /backup.sh   # 06:00 todos os dias
0 6 * * 1       /relatorio.sh # 06:00 toda segunda
0 0 1 * *       /mensal.sh   # Meia-noite dia 1 de cada mês
*/5 * * * *     /monitor.sh  # A cada 5 minutos
30 2 * * 0,6    /manutencao.sh# 02:30 sábados e domingos

# Atalhos especiais:
@reboot  /inicializacao.sh  # Na inicialização do sistema
@daily   /diario.sh          # Uma vez por dia (meia-noite)
@weekly  /semanal.sh         # Uma vez por semana (dom, meia-noite)
@monthly /mensal.sh          # Uma vez por mês
⚙️ GERENCIANDO CRONTAB
# Editar crontab do usuário atual
$ crontab -e

# Listar tarefas do usuário atual
$ crontab -l

# Crontab do root (via sudo)
$ sudo crontab -e
$ sudo crontab -l -u root

# Exemplo prático: backup diário às 02:00
$ crontab -e
# Adicionar a linha:
0 2 * * * /usr/local/bin/backup.sh >> /var/log/backup_cron.log 2>&1
# >> = append ao log  2>&1 = stderr junto com stdout

# Crontab do sistema (com usuário especificado)
$ sudo nano /etc/cron.d/meu-agendamento
SHELL=/bin/bash
PATH=/usr/local/sbin:/usr/local/bin:/sbin:/bin:/usr/sbin:/usr/bin
# m  h  dom mon dow  user     command
0 3 * * *  root  /usr/local/bin/limpeza.sh
🔄 ANACRON — PARA SISTEMAS QUE FICAM DESLIGADOS
$ cat /etc/anacrontab
# period  delay  job-identifier  command
1         5      cron.daily      run-parts /etc/cron.daily
7         10     cron.weekly     run-parts /etc/cron.weekly
@monthly  15     cron.monthly    run-parts /etc/cron.monthly
# period  = frequência em dias (1=diário, 7=semanal)
# delay   = atraso em minutos após ligar antes de executar
# Garante que tarefas "perdidas" sejam executadas
⚠️ PATH no cron: O cron usa um PATH mínimo. Sempre use caminhos absolutos nos scripts agendados: /usr/bin/python3 em vez de python3.

Sistema de Monitoramento Automatizado
⏱️ 30 min · Dificuldade: ⭐⭐⭐⭐
+75 XP
▼
🎯 Criar script completo de monitoramento com alertas, log e agendamento via cron
📊 SCRIPT DE MONITORAMENTO COMPLETO
#!/bin/bash
# monitoramento.sh — Verifica saúde do sistema
# Agendar: */5 * * * * /usr/local/bin/monitoramento.sh

set -uo pipefail

LOG="/var/log/monitoramento.log"
ALERTA_LOG="/var/log/alertas.log"

# Limites de alerta
LIMITE_CPU=80
LIMITE_MEM=85
LIMITE_DISCO=90

log()   { echo "[$(date '+%Y-%m-%d %H:%M:%S')] [OK]    $*" >> "$LOG"; }
alerta(){ echo "[$(date '+%Y-%m-%d %H:%M:%S')] [ALERTA] $*" | tee -a "$LOG" "$ALERTA_LOG"; }

# ── CPU ──────────────────────────────────────
CPU_IDLE=$(top -bn1 | grep "Cpu(s)" | awk '{print $8}' | tr -d '%')
CPU_USO=$(echo "100 - $CPU_IDLE" | bc | cut -d. -f1)
if [ "$CPU_USO" -ge "$LIMITE_CPU" ]; then
    alerta "CPU em ${CPU_USO}% (limite: ${LIMITE_CPU}%)"
    # Top 3 processos consumindo CPU
    ps aux --sort=-%cpu | awk 'NR>1 && NR<=4 {print "  ",$1,$3"%",$11}' >> "$ALERTA_LOG"
else
    log "CPU: ${CPU_USO}%"
fi

# ── Memória ───────────────────────────────────
MEM_TOTAL=$(free -m | awk 'NR==2{print $2}')
MEM_USADA=$(free -m | awk 'NR==2{print $3}')
MEM_PCT=$(echo "scale=0; $MEM_USADA*100/$MEM_TOTAL" | bc)
if [ "$MEM_PCT" -ge "$LIMITE_MEM" ]; then
    alerta "Memória em ${MEM_PCT}% (${MEM_USADA}MB/${MEM_TOTAL}MB)"
else
    log "Memória: ${MEM_PCT}% (${MEM_USADA}MB)"
fi

# ── Disco ─────────────────────────────────────
while IFS= read -r linha; do
    USO=$(echo "$linha" | awk '{print $5}' | tr -d '%')
    MOUNT=$(echo "$linha" | awk '{print $6}')
    if [ "$USO" -ge "$LIMITE_DISCO" ]; then
        alerta "Disco $MOUNT em ${USO}%!"
    fi
done < <(df -h | grep "^/dev")

# ── Serviços críticos ─────────────────────────
SERVICOS_CRITICOS=("ssh" "cron")
for SVC in "${SERVICOS_CRITICOS[@]}"; do
    if ! systemctl is-active --quiet "$SVC"; then
        alerta "SERVIÇO DOWN: $SVC — tentando reiniciar..."
        systemctl restart "$SVC" 2>> "$LOG" && log "$SVC reiniciado"
    fi
done

# Agendar a cada 5 minutos:
$ sudo cp monitoramento.sh /usr/local/bin/
$ sudo chmod +x /usr/local/bin/monitoramento.sh
$ sudo crontab -e
# Adicionar: */5 * * * * /usr/local/bin/monitoramento.sh
✅ Checklist final: set -euo pipefail ativo · trap configurado · log com timestamps · variáveis validadas · crontab agendado · monitoramento funcionando · alertas em arquivo separado
