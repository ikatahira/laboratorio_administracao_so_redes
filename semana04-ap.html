Glossário de Access Points
SSID
Service Set Identifier — nome da rede Wi-Fi. Pode ser broadcast ou oculto.
WPA3-SAE
Simultaneous Authentication of Equals — substitui PSK, resistente a ataques de dicionário.
RADIUS
Remote Authentication Dial-In User Service — autenticação centralizada para redes corporativas.
Roaming
Transição de cliente entre APs sem perda de conexão. 802.11r acelera o processo.
WPS
Wi-Fi Protected Setup — protocolo com vulnerabilidade grave (PIN brute-force). SEMPRE desativar.
OFDMA
Orthogonal Frequency Division Multiple Access — permite múltiplos clientes simultâneos no Wi-Fi 6.

📦 Atenção — Todos os SOs estão em Máquinas Virtuais (VirtualBox)

As configurações de rede a seguir são realizadas dentro das VMs no VirtualBox. Neste laboratório, os computadores de mesa se conectam ao WG302 por cabo Ethernet (RJ-45) — não por Wi-Fi. O Wi-Fi do AP é a rede que os clientes externos (celulares, notebooks) vão usar; o seu PC de mesa é o administrador, conectado com fio.

Para que a VM enxergue o AP através da placa Ethernet do PC host, o adaptador de rede da VM deve estar em modo Bridge (Modo Ponte) — caso contrário a VM ficará isolada na rede virtual do VirtualBox e não verá o AP nem a rede física do laboratório.

Como verificar e configurar o modo Bridge no VirtualBox
1
Antes de ligar a VM, confirme que o cabo Ethernet do PC de mesa está conectado à porta LAN do WG302 (ou ao switch que o WG302 está ligado).
2
Com a VM desligada, clique nela com botão direito no VirtualBox → Configurações (Settings).
3
No menu lateral clique em Rede (Network). Na aba Adaptador 1, certifique-se de que Habilitar Adaptador de Rede está marcado.
4
Em "Conectado a:" selecione Placa em modo Bridge (Bridged Adapter). Em "Nome:" escolha a placa Ethernet (cabeada) do PC host — geralmente listada como "Realtek", "Intel Ethernet" ou similar. Não selecione Wi-Fi, pois o PC usa cabo.
5
Clique em OK e inicie a VM. A VM agora enxerga a rede física do AP como se fosse um computador independente ligado por cabo naquela rede.
6
Para confirmar que o Bridge está funcionando: dentro da VM, abra o Prompt de Comando e execute ipconfig. O IP exibido deve estar na mesma sub-rede do AP (ex: 192.168.0.X ou 192.168.10.X). Se aparecer 192.168.56.X, a rede ainda está em modo Host-Only — revise o passo 4.
⚠️ Resumo da topologia do lab: PC de mesa → cabo Ethernet RJ-45 → porta LAN do WG302. Dentro do PC roda a VM (VirtualBox, modo Bridge na placa Ethernet). A VM acessa a interface web do AP pelo IP 192.168.0.228. O Wi-Fi (SSID FATEC-LAB-S4) é a rede transmitida pelo AP para dispositivos sem fio externos — não é usada pelo PC de mesa.
💡 Contexto: Neste lab, o PC de mesa está conectado ao WG302 por cabo Ethernet. Dentro do PC, a VM (VirtualBox em modo Bridge) acessa o AP pela placa Ethernet do host. A configuração de IP é feita pelo adaptador com fio (LAN) da VM — não há placa Wi-Fi envolvida do lado do administrador. O Windows XP gerencia conexões com fio pelo adaptador de rede padrão; verifique se o driver de rede está instalado.

Deve aparecer "Adaptador Ethernet" com endereço físico (MAC)

⚠️ Se o ícone não aparecer ou mostrar X vermelho, o driver da placa de rede não está instalado ou o cabo não está conectado. Verifique o cabo RJ-45 e instale o driver correto (ex: Realtek, Intel, VirtualBox Ethernet Adapter).

Abrir Propriedades da Conexão de Área Local

Método 2 — Obter IP Automático via DHCP (se o AP fornecer)
Se o WG302 estiver configurado com servidor DHCP ativo, a VM pode obter IP automaticamente sem configuração manual.
1
Abrir Propriedades TCP/IP da Conexão de Área Local

2
Habilitar obtenção automática de IP

Confirmar e acessar o AP

Resolução de Problemas
❌ Não consigo acessar 192.168.0.228
Verifique o cabo RJ-45 (LEDs no AP e na placa de rede devem piscar). Confirme que o IP do PC está em 192.168.0.X. Verifique o modo Bridge no VirtualBox. Desative o firewall do Windows XP temporariamente.
⚠️ Senha incorreta / falha na autenticação
Confirme o tipo de segurança (WPA vs WEP). Verifique se o SSID cadastrado manualmente está exatamente igual — diferencia maiúsculas.
⚠️ Conectado mas sem internet
Confirme IP correto com ipconfig. Tente ping 8.8.8.8. Verifique se o gateway e DNS estão configurados. Libere/renove IP com ipconfig /release e ipconfig /renew.
🔧 Sinal fraco ou instável
Reduza obstáculos entre o dispositivo e o WG302. Troque o canal do AP (use 1, 6 ou 11). Ajuste a posição das antenas para posição vertical.

Comandos úteis — Terminal Windows 11
── Verificar adaptador Ethernet e IP ────────────
PS> ipconfig /all (detalhes de todos os adaptadores)

── Renovar IP via DHCP ───────────────────────────
PS> ipconfig /release
PS> ipconfig /renew

── Testar conectividade com o AP ─────────────────
PS> ping 192.168.0.228 (deve responder se cabo e IP estiverem corretos)
PS> tracert 192.168.0.228 (ver saltos até o AP)

── Reset completo de rede (executar como admin) ──
PS> netsh winsock reset
PS> netsh int ip reset
→ Reiniciar a VM após o reset
Resolução de Problemas — Windows 11
❌ Não acessa 192.168.0.228
Verifique o cabo RJ-45 (LEDs no AP e na placa de rede). Confirme IP 192.168.0.X no adaptador Ethernet via ncpa.cpl. Verifique modo Bridge no VirtualBox (placa Ethernet, não Wi-Fi do host).
⚠️ IP 169.254.x.x
Sem resposta DHCP — configure IP fixo: 192.168.0.10 / 255.255.255.0 / gateway 192.168.0.228. Verifique modo Bridge no VirtualBox — certifique-se que "Nome:" aponta para a placa Ethernet física do host.
⚠️ Adaptador Ethernet sem link
No VirtualBox confirme Adaptador 1 → Bridge → Nome = placa Ethernet do host. Reinicie a VM após trocar o modo. Verifique se o cabo está bem encaixado nos dois lados.

Abra o navegador e acesse http://192.168.0.228 → login admin / password. Configure os SSIDs pelo menu Wireless:
