Glossário de VLANs
VLAN
Virtual LAN — segmenta logicamente a rede sem hardware adicional, isolando tráfego por função.
Trunk Port
Porta que carrega tráfego de múltiplas VLANs usando tags IEEE 802.1Q. Conecta switches entre si.
Access Port
Porta que pertence a uma única VLAN. Conecta dispositivos finais (PCs, impressoras).
Native VLAN
VLAN sem tag em links trunk. Frames não tagueados pertencem à native VLAN (default: VLAN 1).
STP
Spanning Tree Protocol — evita loops de layer 2 bloqueando portas redundantes automaticamente.
Port Security
Limita MACs por porta, bloqueando dispositivos não autorizados. Ativa shutdown na violação.

Configuração de VLANs no Switch
🔀 Switches · ⏱ 30 min · Dificuldade: Médio
+60 XP ▼
Criar 3 VLANs (Administração, Vendas, TI) e configurar portas de acesso e trunk.
1

Acesse o switch via console com enable, depois configure terminal.

2

Crie as três VLANs no switch:

Switch(config)# vlan 10
Switch(config-vlan)# name Administracao
Switch(config-vlan)# exit
Switch(config)# vlan 20
Switch(config-vlan)# name Vendas
Switch(config-vlan)# exit
Switch(config)# vlan 30
Switch(config-vlan)# name TI
3

Atribua porta Fa0/1 à VLAN 10 (Administração) como access port:

Switch(config)# interface FastEthernet 0/1
Switch(config-if)# switchport mode access
Switch(config-if)# switchport access vlan 10
! Repita para Fa0/2 (VLAN 20) e Fa0/3 (VLAN 30)
4

Configure a porta Gi0/1 como trunk para o roteador:

Switch(config)# interface GigabitEthernet 0/1
Switch(config-if)# switchport mode trunk
Switch(config-if)# switchport trunk allowed vlan 10,20,30
5

Verifique com show vlan brief e show interfaces trunk.
