📝 Configuração da VM no VirtualBox

# Parâmetros recomendados para a VM do Ubuntu Desktop Nome: Ubuntu-Desktop-22 Tipo: Linux | Versão: Ubuntu (64-bit) RAM: 4096 MB (mínimo 2048 MB) Disco: 25 GB VDI dinâmico Rede: Adaptador 1 → NAT

📝 Pós-Instalação

# Atualizar o sistema após instalação sudo apt update && sudo apt upgrade -y # Instalar utilitários básicos sudo apt install net-tools curl wget htop -y # Verificar versão instalada lsb_release -a
💡 DICA: Após a instalação e o update, tire um Snapshot no VirtualBox (Máquina → Tirar Snapshot) para salvar o estado limpo da VM. Isso permite reverter facilmente em caso de problemas.

📝 Configuração da VM no VirtualBox

# Parâmetros recomendados para a VM do Ubuntu Server Nome: Ubuntu-Server-22 Tipo: Linux | Versão: Ubuntu (64-bit) RAM: 1024 MB (servidor não precisa de GUI) Disco: 20 GB VDI dinâmico Rede: Adaptador 1 → NAT | Adaptador 2 → Rede Interna (intnet)

📝 Verificação do LVM após instalação

# Verificar volumes LVM criados pelo instalador sudo pvs sudo vgs sudo lvs # Expandir LV para usar todo o espaço disponível no VG sudo lvextend -l +100%FREE /dev/ubuntu-vg/ubuntu-lv sudo resize2fs /dev/ubuntu-vg/ubuntu-lv # Verificar status do SSH sudo systemctl status ssh # Verificar IP da VM para acesso remoto ip addr show | grep inet
💡 DICA: Na tela de tipo de instalação, selecione Ubuntu Server (minimized) — não a opção padrão. Isso garante o menor conjunto de pacotes, reduzindo a superfície de ataque. Use comandos de verificação como sudo lvs, df -h e systemctl status ssh para confirmar as configurações.

📝 IP Forwarding e Masquerading

# Habilitar IP forwarding permanentemente sudo nano /etc/sysctl.conf # Descomentar a linha: net.ipv4.ip_forward=1 sudo sysctl -p # Verificar que está ativo (deve retornar 1) cat /proc/sys/net/ipv4/ip_forward

📝 Regras UFW

# SEMPRE libere SSH antes de ativar o UFW sudo ufw allow 22/tcp # Ativar o firewall sudo ufw enable # Definir políticas padrão sudo ufw default deny incoming sudo ufw default allow outgoing sudo ufw default allow routed # Regras para serviços permitidos sudo ufw allow 80/tcp # HTTP sudo ufw allow 443/tcp # HTTPS # Bloquear serviços inseguros sudo ufw deny 23/tcp # Telnet sudo ufw deny 21/tcp # FTP # Verificar status sudo ufw status verbose
💡 DICA: Sempre verifique os resultados de cada comando antes de prosseguir. Use sudo ufw status verbose, ip route show e ping -c 3 8.8.8.8 nas outras VMs para confirmar que o masquerading e o IP forwarding estão funcionando corretamente.
