🔄 Precisa reinstalar? Siga este passo a passo:
  1. Abra o VirtualBox e selecione a máquina desejada

✏️ Exercício 3: Configuração do Firewall UFW como Gateway

3
VM Firewall — Ubuntu Server + UFW como Gateway

📋 Objetivo: Configurar VM dedicada com Ubuntu Server como firewall/gateway da rede interna

⏱️ Duração: 45 minutos

📄 Descrição: Criar VM com 2 NICs, habilitar IP forwarding, masquerading e aplicar regras UFW

💾
ISO para instalação:

A ISO do Ubuntu Firewall é a mesma utilizada para a instalação do Ubuntu Server. Você não precisa baixar — todas as ISOs estão disponíveis na unidade Z: (Prof. Isaque). Basta apontá-la no VirtualBox ao criar esta VM.

⚙️ Pré-requisito: Configuração de Rede no VirtualBox

Antes de instalar o Ubuntu Firewall, é necessário configurar corretamente os adaptadores de rede da VM no VirtualBox:

  1. Acesse as Configurações da VM UbuntuFirewall no VirtualBox
  2. Vá em Rede → Adaptador 1
  3. Defina "Conectado a" como Rede Interna e o nome como Intranet
  4. Clique na aba Adaptador 2 e desabilite a placa de rede (desmarque "Habilitar Placa de Rede")
  5. Clique em OK para salvar

✅ Adaptador 1 — Rede Interna (Intranet)

🚫 Adaptador 2 — Desabilitar a placa de rede

🔧 A placa de rede não funcionou após a instalação? Siga estes passos:

✅ 1. Ativar a interface manualmente

Se a interface de rede estiver inativa, ative-a com:

sudo ip link set enp0s3 up

✅ 2. Solicitar IP via DHCP

Após ativar a interface, solicite um endereço IP automaticamente:

sudo dhclient enp0s3

🧪 3. Testar a conectividade

Verifique se há conexão com a internet:

ping -c 4 8.8.8.8

Se responder, teste também a resolução de nomes (DNS):

ping -c 4 google.com
⚠️ Atenção: Substitua enp0s3 pelo nome real da sua interface, caso seja diferente. Use ip link show para verificar o nome correto.
