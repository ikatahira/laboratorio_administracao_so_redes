ISO-103 · SEMANA 5.1

Saída pela Internet via Firewall (Gateway NAT)

Servidor + Desktop roteando tráfego pelo firewall da Semana 5.2
← Índice

🎯 Objetivo da Aula

Nesta aula você vai configurar o ubuntu-server e o ubuntu-desktop para que ambos consigam acessar a internet usando o firewall como gateway NAT (configurado na Semana 5.2). Ao final, um simples ping 8.8.8.8 em ambas as máquinas deve funcionar — tráfego saindo pela VM de firewall.

T
Topologia da Rede

Toda saída para internet passa pelo firewall (enp0s8 → enp0s3 → NAT VirtualBox)

📌 Lembre-se: o firewall (Semana 5.2) já tem IP Forwarding ativo e regras NAT (masquerading) configuradas. O que fazemos aqui é dizer para o servidor e para o desktop: "quando precisar sair para a internet, use o firewall como gateway".

  • ✓
    A VM ubuntu-server está com a placa de rede em modo Host-Only (mesma rede interna)
  • ✓
    A VM ubuntu-desktop está com a placa de rede em modo Host-Only (mesma rede interna)

1
Configurar o Ubuntu Server
🖥 Contexto: Faça login na VM ubuntu-server com usuário ubuntu-server e senha Fatec2026. Vamos configurar o IP estático e o gateway via Netplan.
1
Verificar o nome da interface de rede
Precisamos saber o nome exato da placa (pode ser enp0s3, enp0s8, eth0…)

Execute o comando abaixo e anote o nome da interface conectada à rede Host-Only:

ubuntu-server@server:~$ ip addr show
Saída esperada (exemplo)
2: enp0s3: <BROADCAST,MULTICAST,UP,LOWER_UP> mtu 1500 qdisc fq_codel state UP inet 192.168.56.10/24 brd 192.168.56.255 scope global enp0s3
💡 Dica: Se aparecer enp0s3, enp0s8 ou eth0, use esse nome nos próximos passos. Aqui usaremos enp0s3 como exemplo.
2
Editar o arquivo Netplan
Configurar IP estático + gateway apontando para o firewall

Abra o arquivo de configuração do Netplan:

ubuntu-server@server:~$ sudo nano /etc/netplan/00-installer-config.yaml

Substitua o conteúdo pelo seguinte (cuidado com a indentação — use espaços, nunca Tab):

network: version: 2 ethernets: enp0s3: # ← troque pelo nome da SUA interface addresses: - 192.168.56.10/24 # IP estático do servidor routes: - to: default via: 192.168.56.1 # ← IP do firewall (gateway) nameservers: addresses: - 8.8.8.8 # DNS Google - 1.1.1.1 # DNS Cloudflare
⚠️ Atenção: No Netplan, cada nível de indentação usa 2 espaços. Um erro de espaçamento faz o arquivo ser rejeitado. Verifique com cuidado antes de salvar.

Para salvar no nano: pressione Ctrl+X → Y → Enter.

3
Aplicar as configurações
Ativar o Netplan e verificar se o gateway foi aplicado
ubuntu-server@server:~$ sudo netplan apply # aguarde alguns segundos ubuntu-server@server:~$ ip route show
Saída esperada
default via 192.168.56.1 dev enp0s3 proto static 192.168.56.0/24 dev enp0s3 proto kernel scope link src 192.168.56.10
✅ Sucesso: A linha default via 192.168.56.1 confirma que o firewall é o gateway padrão. Todo tráfego destinado à internet vai passar por ele.
4
Testar a saída pela internet
Ping para o DNS do Google e teste de resolução de nomes
ubuntu-server@server:~$ ping -c 4 8.8.8.8 # testa conectividade IP ubuntu-server@server:~$ ping -c 4 google.com # testa resolução de nomes (DNS) ubuntu-server@server:~$ curl -s ifconfig.me # mostra o IP público de saída
Saída esperada — ping
PING 8.8.8.8 (8.8.8.8) 56(84) bytes of data. 64 bytes from 8.8.8.8: icmp_seq=1 ttl=117 time=12.4 ms 64 bytes from 8.8.8.8: icmp_seq=2 ttl=117 time=11.9 ms 4 packets transmitted, 4 received, 0% packet loss
❌ Se o ping falhar: verifique se o firewall está ligado (sudo ufw status no firewall deve mostrar active) e se o IP Forwarding está ativo (cat /proc/sys/net/ipv4/ip_forward deve retornar 1).
2
Configurar o Ubuntu Desktop
🖥 Contexto: Faça login na VM ubuntu-desktop com usuário ubuntu-desktop e senha Fatec2026. No Desktop, podemos configurar via interface gráfica OU via Netplan (terminal). Veja as duas opções:
1
Verificar interface de rede
Abra um terminal no Desktop (Ctrl+Alt+T)
ubuntu-desktop@desktop:~$ ip addr show # anote o nome da interface
2
Editar o Netplan do Desktop
IP estático 192.168.56.20 com gateway apontando para o firewall
ubuntu-desktop@desktop:~$ sudo nano /etc/netplan/01-network-manager-all.yaml
💡 Dica: No Ubuntu Desktop o arquivo pode se chamar 01-network-manager-all.yaml. Use ls /etc/netplan/ para confirmar o nome.

Conteúdo a inserir:

network: version: 2 renderer: NetworkManager ethernets: enp0s3: # ← troque pelo nome da SUA interface addresses: - 192.168.56.20/24 # IP estático do desktop routes: - to: default via: 192.168.56.1 # ← IP do firewall (gateway) nameservers: addresses: - 8.8.8.8 - 1.1.1.1
3
Aplicar e testar
ubuntu-desktop@desktop:~$ sudo netplan apply ubuntu-desktop@desktop:~$ ip route show ubuntu-desktop@desktop:~$ ping -c 4 8.8.8.8
Saída esperada — rota
default via 192.168.56.1 dev enp0s3 proto static 192.168.56.0/24 dev enp0s3 proto kernel scope link src 192.168.56.20
🖱️ Configurar via Configurações de Rede (GUI)
1
Abrir Configurações → Rede

Clique no ícone de rede na barra superior → Configurações de Rede (ou acesse via Configurações do Sistema → Rede).

2
Editar a conexão com fio

Clique na engrenagem ⚙️ ao lado da conexão com fio ativa.

3
Aba IPv4 → Modo Manual

Preencha os campos:

Endereço: 192.168.56.20 Máscara: 255.255.255.0 (ou /24) Gateway: 192.168.56.1 (← IP do firewall) DNS: 8.8.8.8, 1.1.1.1

Clique em Aplicar e reconecte a interface.

⚠️ Atenção: Após aplicar via GUI, verifique no terminal com ip route show se o gateway 192.168.56.1 aparece como rota padrão.
3
Teste Final — Ambas as Máquinas na Internet
🔬 Momento da verdade: Execute estes testes em ambas as VMs para confirmar que tudo funciona corretamente.
🧪 Bateria de Testes Completa
🖥 ubuntu-server — execute em cada VM
# Teste 1 — Conectividade com o firewall (gateway) $ ping -c 2 192.168.56.1 # Teste 2 — Saída para internet (IP) $ ping -c 4 8.8.8.8 # Teste 3 — Resolução de nomes (DNS) $ ping -c 2 google.com # Teste 4 — Verificar rota padrão $ ip route show | grep default # Teste 5 — Traceroute (mostra o caminho) $ traceroute -m 5 8.8.8.8 # ↑ primeiro hop deve ser 192.168.56.1 (o firewall!)
Resultado esperado — traceroute
traceroute to 8.8.8.8 (8.8.8.8), 5 hops max 1 192.168.56.1 1.2 ms ← firewall (gateway interno) 2 10.0.2.2 2.1 ms ← gateway VirtualBox (NAT) 3 ... x.x ms ← roteadores da internet
✅ Prova de conceito: Se o primeiro hop do traceroute é 192.168.56.1, confirma que o tráfego está passando pelo firewall antes de sair para a internet. Missão cumprida!
🔧 Troubleshooting — O que fazer se não funcionar
!
Ping para 8.8.8.8 falha — sem resposta

Verifique no firewall:

# No FIREWALL — verificar IP Forwarding fw$ cat /proc/sys/net/ipv4/ip_forward # deve retornar: 1 # No FIREWALL — verificar regras NAT fw$ sudo iptables -t nat -L -n --line-numbers # deve ter uma linha MASQUERADE para a rede 192.168.56.0/24 # No FIREWALL — verificar status UFW fw$ sudo ufw status verbose
!
Ping para IP funciona mas DNS não resolve
# Testar resolução de DNS manualmente $ nslookup google.com 8.8.8.8 # Verificar /etc/resolv.conf $ cat /etc/resolv.conf # deve conter: nameserver 8.8.8.8 # Liberar porta 53 no firewall (se bloqueada) fw$ sudo ufw allow out 53/udp
!
Rota padrão não aparece ou está errada
# Verificar o arquivo netplan $ cat /etc/netplan/00-installer-config.yaml # Testar sintaxe do netplan $ sudo netplan try # se der erro, ele reverte automaticamente em 120s # Adicionar rota manualmente (temporário — para testar) $ sudo ip route add default via 192.168.56.1
