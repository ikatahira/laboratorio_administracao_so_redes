01
🌐 Configurar IP Estático com Netplan
⏱️ 20 min · Configuração permanente e segura
+60 XP
▼
🎯 Configure um IP estático no Ubuntu Server usando Netplan. IPs estáticos são essenciais para servidores — você sempre sabe onde encontrá-los!
# PASSO 1 — Identificar nome da interface de rede admin@servidor:~$ ip addr show 1: lo: <LOOPBACK> ... 2: enp0s3: <BROADCAST,MULTICAST,UP> mtu 1500 inet 10.0.2.15/24 brd 10.0.2.255 scope global dynamic enp0s3 3: enp0s8: <BROADCAST,MULTICAST,UP> mtu 1500 inet 192.168.56.10/24 ... # Sua interface pode ser: enp0s3, ens33, eth0, etc.
# PASSO 2 — Verificar arquivo Netplan existente admin@servidor:~$ ls /etc/netplan/ 00-installer-config.yaml admin@servidor:~$ cat /etc/netplan/00-installer-config.yaml network: version: 2 ethernets: enp0s3: dhcp4: true
# PASSO 3 — Editar para IP estático (substitua enp0s3 pela sua interface) admin@servidor:~$ sudo nano /etc/netplan/00-installer-config.yaml ## Cole o conteúdo abaixo (atenção aos espaços — YAML é sensível!): network: version: 2 ethernets: enp0s3: dhcp4: false addresses: - 192.168.56.10/24 routes: - to: default via: 192.168.56.1 nameservers: addresses: [8.8.8.8, 8.8.4.4] ## Salvar: Ctrl+X → Y → Enter
# PASSO 4 — Aplicar configuração admin@servidor:~$ sudo netplan try Do you want to keep these settings? [yes]: yes # "try" testa por 120 segundos antes de confirmar — seguro! admin@servidor:~$ sudo netplan apply admin@servidor:~$ ip addr show enp0s3 inet 192.168.56.10/24 brd 192.168.56.255 scope global enp0s3 ✓ IP estático configurado! Vai persistir após reinicialização. # Testar conectividade admin@servidor:~$ ping -c 3 8.8.8.8 3 packets transmitted, 3 received, 0% packet loss
✅ IP estático configurado via Netplan! O servidor sempre terá o mesmo endereço.
02
🔍 Diagnóstico de Rede com Ferramentas
⏱️ 25 min · Localizar e resolver problemas de conectividade
+60 XP
▼
🎯 Domine as ferramentas de diagnóstico que todo administrador de redes precisa conhecer. Cada ferramenta resolve um tipo específico de problema.
# FERRAMENTA 1 — ip: substituto moderno do ifconfig admin@servidor:~$ ip addr show # lista interfaces e IPs admin@servidor:~$ ip route show # tabela de roteamento admin@servidor:~$ ip link show # status físico das interfaces admin@servidor:~$ ip neigh show # tabela ARP (quem está na rede) # FERRAMENTA 2 — ss: substituto moderno do netstat admin@servidor:~$ ss -tuln # portas abertas (-t tcp, -u udp, -l listen, -n numeric) Netid State Local Address:Port tcp LISTEN 0.0.0.0:22 ← SSH escutando admin@servidor:~$ ss -tulnp # -p mostra qual processo usa cada porta
# FERRAMENTA 3 — ping: testar conectividade básica admin@servidor:~$ ping -c 4 8.8.8.8 # teste internet (-c = count) admin@servidor:~$ ping -c 4 google.com # teste DNS + internet # FERRAMENTA 4 — traceroute: ver o caminho até o destino admin@servidor:~$ sudo apt install traceroute -y admin@servidor:~$ traceroute 8.8.8.8 1 192.168.56.1 1.2ms ← gateway local 2 10.0.0.1 3.4ms ← roteador ISP 3 172.16.0.1 5.6ms 4 8.8.8.8 12.3ms ← destino! # Se trava em algum salto, o problema está naquele ponto!
# FERRAMENTA 5 — nmap: verificar portas abertas admin@servidor:~$ sudo apt install nmap -y admin@servidor:~$ nmap -sV localhost PORT STATE SERVICE VERSION 22/tcp open ssh OpenSSH 8.9 # PROCESSO DE TROUBLESHOOTING (sempre nesta ordem!): # 1. Interface up? → ip link show # 2. Tem IP? → ip addr show # 3. Tem gateway? → ip route show # 4. Alcança gateway? → ping <gateway> # 5. Alcança internet? → ping 8.8.8.8 # 6. DNS funciona? → ping google.com
✅ Ferramentas dominadas! Siga sempre a sequência de troubleshooting da camada física até a camada de aplicação.
03
🖧 Instalar e Configurar Servidor DHCP
⏱️ 35 min · ISC DHCP Server com pools e reservas
+65 XP
▼
🎯 Configure um servidor DHCP completo que distribui IPs automaticamente para a rede e reserva endereços fixos para dispositivos específicos.
# PASSO 1 — Instalar o servidor DHCP admin@servidor:~$ sudo apt install isc-dhcp-server -y # PASSO 2 — Definir qual interface o DHCP vai escutar admin@servidor:~$ sudo nano /etc/default/isc-dhcp-server ## Encontre a linha INTERFACESv4 e altere para sua interface: INTERFACESv4="enp0s8"
# PASSO 3 — Editar o arquivo principal de configuração admin@servidor:~$ sudo nano /etc/dhcp/dhcpd.conf ## Adicione ao final do arquivo: subnet 192.168.56.0 netmask 255.255.255.0 { range 192.168.56.100 192.168.56.200; # pool de IPs disponíveis option routers 192.168.56.1; # gateway padrão option domain-name-servers 8.8.8.8, 8.8.4.4; # DNS option domain-name "lab.local"; default-lease-time 600; # 10 minutos max-lease-time 7200; # 2 horas # Reserva de IP para o servidor web (pelo MAC address) host webserver { hardware ethernet 08:00:27:aa:bb:cc; fixed-address 192.168.56.50; } }
# PASSO 4 — Iniciar e habilitar o serviço admin@servidor:~$ sudo systemctl start isc-dhcp-server admin@servidor:~$ sudo systemctl enable isc-dhcp-server admin@servidor:~$ sudo systemctl status isc-dhcp-server ● isc-dhcp-server.service - ISC DHCP IPv4 server Active: active (running) # PASSO 5 — Verificar concessões ativas (clientes que receberam IP) admin@servidor:~$ cat /var/lib/dhcp/dhcpd.leases lease 192.168.56.101 { starts 3 2024/01/15 10:00:00; ends 3 2024/01/15 10:10:00; hardware ethernet 08:00:27:xx:xx:xx; client-hostname "desktop-aluno"; }
✅ Servidor DHCP ativo! Qualquer VM na rede Host-Only receberá IP automaticamente no range 192.168.56.100-200.
04
🔧 Troubleshooting Avançado de Rede
⏱️ 25 min · Resolver problemas comuns sistematicamente
+65 XP
▼
🎯 Resolva cenários de problemas reais de rede seguindo a metodologia de troubleshooting camada por camada.
# CENÁRIO 1 — "Não consigo acessar a internet" admin@servidor:~$ ip addr show # 1. Tem IP? admin@servidor:~$ ip route show # 2. Tem rota padrão? admin@servidor:~$ ping -c 2 192.168.56.1 # 3. Alcança gateway? admin@servidor:~$ ping -c 2 8.8.8.8 # 4. Alcança internet por IP? admin@servidor:~$ ping -c 2 google.com # 5. DNS funciona? admin@servidor:~$ cat /etc/resolv.conf # 6. Verificar DNS configurado
# CENÁRIO 2 — "Serviço está escutando mas não consigo conectar" admin@servidor:~$ ss -tulnp | grep 80 # Apache rodando? admin@servidor:~$ sudo ufw status # Firewall bloqueando porta 80? admin@servidor:~$ sudo ufw allow 80/tcp # Liberar se necessário admin@servidor:~$ curl -I http://localhost # Testar resposta HTTP # CENÁRIO 3 — "Interface caiu" admin@servidor:~$ ip link show enp0s3 enp0s3: <BROADCAST,MULTICAST> state DOWN ← interface DOWN! admin@servidor:~$ sudo ip link set enp0s3 up # Subir a interface admin@servidor:~$ sudo dhclient enp0s3 # Obter IP via DHCP # Ver logs de rede em tempo real admin@servidor:~$ sudo journalctl -f -u NetworkManager
✅ Metodologia de troubleshooting dominada! Sempre siga a sequência: física → IP → rota → DNS → aplicação.
