1. O QUE É UM USUÁRIO NO LINUX?
Pense no Linux como um prédio de apartamentos. Cada morador tem sua chave própria (nome de usuário + senha) que dá acesso somente ao seu apartamento e às áreas comuns. O porteiro-chefe (o usuário root) tem a chave mestre que abre tudo.
🏢
Usuário Comum
joao, maria, pedro
Só acessa sua própria pasta e arquivos com permissão pública. Não pode mexer no sistema.
👮
Superusuário
root
Acesso total ao sistema. Pode criar, apagar e modificar qualquer arquivo. Usar com responsabilidade!
🤖
Usuário de Serviço
www-data, mysql
Criado automaticamente por programas (Apache, MySQL). Não faz login. É dono dos arquivos do programa.
👥
Grupo
ti, rh, devs
Uma coleção de usuários. Facilita dar a mesma permissão para várias pessoas ao mesmo tempo.

▶ Experimente: descubra quem você é no sistema

ubuntu-server — bash
aluno@ubuntu:~$ whoami
# Mostra o nome do usuário atual
aluno@ubuntu:~$ id
# Mostra UID, GID e grupos do usuário
aluno@ubuntu:~$ id root
# Mostra informações do root
aluno@ubuntu:~$ whoami
aluno

aluno@ubuntu:~$ id
uid=1000(aluno) gid=1000(aluno) groups=1000(aluno),4(adm),27(sudo)
💡 uid=1000 = identificador único do usuário  |  gid=1000 = grupo principal  |  groups = todos os grupos

aluno@ubuntu:~$ id root
uid=0(root) gid=0(root) groups=0(root)
⚠️ O root sempre tem uid=0. É assim que o Linux sabe quem é o superusuário — pelo número, não pelo nome!
📁
2. ONDE O LINUX GUARDA OS USUÁRIOS?
O Linux não usa banco de dados para guardar usuários — usa arquivos de texto simples! Existem 3 arquivos principais que você precisa conhecer:
/etc/passwd
Lista de todos os usuários. Qualquer um pode ler este arquivo (sem senhas aqui).
/etc/shadow
Senhas criptografadas. Só root pode ler! É aqui que ficam os hashes das senhas.
/etc/group
Lista de todos os grupos e seus membros. Qualquer um pode ler.

▶ Veja o conteúdo real do /etc/passwd

ubuntu-server — bash
aluno@ubuntu:~$ cat /etc/passwd | head -5
# Mostra as 5 primeiras linhas do arquivo de usuários
aluno@ubuntu:~$ cat /etc/passwd | head -5
root:x:0:0:root:/root:/bin/bash
daemon:x:1:1:daemon:/usr/sbin:/usr/sbin/nologin
bin:x:2:2:bin:/bin:/usr/sbin/nologin
sys:x:3:3:sys:/dev:/usr/sbin/nologin
aluno:x:1000:1000:Aluno ISO103:/home/aluno:/bin/bash
Anatomia de uma linha:
aluno : nome de login  |  x : senha está no /etc/shadow  |  1000 : UID  |  1000 : GID  |  Aluno ISO103 : comentário  |  /home/aluno : pasta pessoal  |  /bin/bash : shell padrão
🔒 Tente cat /etc/shadow sem sudo — vai dar erro de permissão. Isso é proposital! O Linux protege os hashes de senha para que nenhum usuário comum possa tentar quebrá-los.
👤
3. CRIANDO SEU PRIMEIRO USUÁRIO
O comando useradd cria usuários. Mas atenção: sozinho ele faz o mínimo. Precisamos de alguns parâmetros para criar um usuário completo que consiga fazer login.
🔧 Os parâmetros mais usados:
-m
Cria a pasta pessoal
/home/nomeusuario
-s /bin/bash
Define o shell padrão
(necessário para login)
-c "Nome"
Nome completo do usuário
(comentário no passwd)
-G grupo
Adiciona a grupos
extras na criação

▶ Crie um usuário chamado "joao" passo a passo

ubuntu-server — bash (como root/sudo)
aluno@ubuntu:~$ sudo useradd -m -s /bin/bash -c "Joao Silva" joao
# -m = cria /home/joao | -s = shell bash | -c = nome completo
aluno@ubuntu:~$ sudo passwd joao
# Define a senha para joao
aluno@ubuntu:~$ grep joao /etc/passwd
# Confirma que o usuário foi criado
aluno@ubuntu:~$ ls -la /home/
# Vê que a pasta /home/joao foi criada
aluno@ubuntu:~$ sudo useradd -m -s /bin/bash -c "Joao Silva" joao
[sem output = deu certo! No Linux, silêncio é sinal de sucesso ✅]

aluno@ubuntu:~$ sudo passwd joao
New password: ••••••••
Retype new password: ••••••••
passwd: password updated successfully

aluno@ubuntu:~$ grep joao /etc/passwd
joao:x:1001:1001:Joao Silva:/home/joao:/bin/bash
✅ Usuário joao criado! UID=1001 (o Linux atribui automaticamente, começando do 1000 para usuários reais)

aluno@ubuntu:~$ ls -la /home/
drwxr-xr-x 4 root root 4096 Apr 24 10:00 .
drwxr-xr-x 2 aluno aluno 4096 Apr 24 09:00 aluno
drwxr-xr-x 2 joao joao 4096 Apr 24 10:00 joao
📁 A pasta /home/joao foi criada automaticamente pelo parâmetro -m!
⚠️ Sem o -m, o usuário é criado mas não tem pasta /home. Sem o -s /bin/bash, ele não consegue fazer login interativo. Sempre use esses dois parâmetros!
👥
4. GRUPOS — PERMISSÕES PARA VÁRIOS DE UMA VEZ
Imagine que você tem 10 funcionários do RH. Em vez de dar permissão ao arquivo para cada um dos 10, você cria um grupo chamado "rh", coloca todos lá, e dá a permissão uma só vez para o grupo. Muito mais prático!

▶ Crie um grupo e adicione o joao

ubuntu-server — bash
aluno@ubuntu:~$ sudo groupadd ti
# Cria o grupo "ti"
aluno@ubuntu:~$ sudo usermod -aG ti joao
# -aG = adiciona joao ao grupo ti (sem remover dos outros grupos)
aluno@ubuntu:~$ id joao
# Verifica os grupos de joao
aluno@ubuntu:~$ grep ti /etc/group
# Vê o grupo "ti" no arquivo de grupos
aluno@ubuntu:~$ sudo groupadd ti
[sem output = grupo criado com sucesso ✅]

aluno@ubuntu:~$ sudo usermod -aG ti joao
[sem output = joao adicionado ao grupo ti ✅]

aluno@ubuntu:~$ id joao
uid=1001(joao) gid=1001(joao) groups=1001(joao),1002(ti)
✅ joao agora está em 2 grupos: o grupo dele próprio (joao) e o grupo ti!

aluno@ubuntu:~$ grep ti /etc/group
ti:x:1002:joao
Formato: nome_grupo : x : GID : membros
💡 Por que -aG e não só -G?
O -G sozinho substitui todos os grupos do usuário. O -aG (append) adiciona ao grupo sem remover os outros. Quase sempre você quer o -aG!
🔐
5. PERMISSÕES — O QUE SIGNIFICA rwx?
Todo arquivo e pasta no Linux tem permissões que dizem quem pode fazer o quê. São 3 letras para 3 tipos de ação:
r = read (ler)  |  w = write (escrever/modificar)  |  x = execute (executar).
E essas 3 ações são definidas para 3 grupos de pessoas: o dono, o grupo e os outros.

▶ Veja as permissões de arquivos reais

ubuntu-server — bash
aluno@ubuntu:~$ ls -la /home/joao/
# -l = formato longo (mostra permissões) | -a = mostra arquivos ocultos
aluno@ubuntu:~$ ls -la /etc/passwd /etc/shadow
# Compara as permissões desses dois arquivos importantes
aluno@ubuntu:~$ ls -la /home/joao/
total 20
drwxr-xr-x 2 joao joao 4096 Apr 24 10:00 .
drwxr-xr-x 4 root root 4096 Apr 24 10:00 ..
-rw-r--r-- 1 joao joao 220 Apr 24 10:00 .bash_logout
-rw-r--r-- 1 joao joao 3526 Apr 24 10:00 .bashrc
-rw-r--r-- 1 joao joao 807 Apr 24 10:00 .profile

aluno@ubuntu:~$ ls -la /etc/passwd /etc/shadow
-rw-r--r-- 1 root root 1823 Apr 24 10:00 /etc/passwd
-rw-r----- 1 root shadow 987 Apr 24 10:00 /etc/shadow
🔍 Decompondo: -rw-r--r--
-
tipo
arquivo
|
rw-
DONO
lê + escreve
r--
GRUPO
só lê
r--
OUTROS
só lê
📌 O traço (-) no lugar de uma letra = sem essa permissão. Ex: r-- = só leitura, sem escrita, sem execução.
⚠️ Repare: -rw-r----- no /etc/shadow significa que outros não têm NENHUM acesso (---). Por isso usuários comuns não conseguem ler as senhas!
⚙️
6. MUDANDO PERMISSÕES COM chmod
O comando chmod (change mode) muda as permissões de um arquivo. Você pode usar de dois jeitos: com letras (mais fácil de lembrar) ou com números (mais rápido).
📊 Tabela de permissões em números (octal):
4
r--
Só leitura
2
-w-
Só escrita
1
--x
Só execução
7
rwx
4+2+1 = tudo
6
rw-
4+2 = ler+escrever
0
---
Nenhum acesso
Exemplo: chmod 755 script.sh
→ Dono=7(rwx)   Grupo=5(r-x)   Outros=5(r-x)
→ O dono faz tudo. Grupo e outros podem ler e executar, mas não modificar.

▶ Veja o chmod em ação — antes e depois

ubuntu-server — bash
# Criar um arquivo de teste
aluno@ubuntu:~$ touch /tmp/teste.txt
aluno@ubuntu:~$ ls -la /tmp/teste.txt
# Veja as permissões padrão que o Linux atribui
aluno@ubuntu:~$ chmod 600 /tmp/teste.txt
# Muda para: dono=rw, grupo=nada, outros=nada
aluno@ubuntu:~$ ls -la /tmp/teste.txt
# Veja a mudança! Também com notação simbólica:
aluno@ubuntu:~$ chmod u+x /tmp/teste.txt
# u=dono, +=adicionar, x=execução
aluno@ubuntu:~$ ls -la /tmp/teste.txt
aluno@ubuntu:~$ touch /tmp/teste.txt
[arquivo criado ✅]

aluno@ubuntu:~$ ls -la /tmp/teste.txt
-rw-r--r-- 1 aluno aluno 0 Apr 24 10:05 /tmp/teste.txt
Permissão inicial: dono lê+escreve, grupo e outros só leem (padrão do Linux)

aluno@ubuntu:~$ chmod 600 /tmp/teste.txt
[sem output = ok ✅]

aluno@ubuntu:~$ ls -la /tmp/teste.txt
-rw------- 1 aluno aluno 0 Apr 24 10:05 /tmp/teste.txt
✅ Agora só o dono pode ler e escrever. Grupo e outros não têm nenhum acesso (—)!

aluno@ubuntu:~$ chmod u+x /tmp/teste.txt
[adicionado bit de execução para o dono ✅]

aluno@ubuntu:~$ ls -la /tmp/teste.txt
-rwx------ 1 aluno aluno 0 Apr 24 10:05 /tmp/teste.txt
✅ Agora tem o x! Dono pode ler, escrever e executar. Equivale ao 700.
📊 Permissões mais comuns — cole isso no seu caderno:
755 = rwxr-xr-x → Diretórios públicos (dono edita, outros acessam)
644 = rw-r--r-- → Arquivos de texto normais
600 = rw------- → Arquivos privados (chaves SSH, configs sensíveis)
700 = rwx------ → Diretórios privados (só o dono entra)
777 = rwxrwxrwx → ⚠️ PERIGOSO — todos fazem tudo!
